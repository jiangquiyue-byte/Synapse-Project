## 当前仍需下一任开发者继续处理的重点

虽然记忆搜索接口已经恢复 200，但**正式 Vercel 环境目前返回的 `backend_label` 仍是 `local-hash-fallback`**。这意味着：

> 当前生产环境的“免费语义记忆”代码路径已经合入，但正式部署环境中的嵌入能力仍未真正稳定跑到 Hugging Face 远程或 FastEmbed 本地语义向量分支。

这不是前端问题，而是**正式部署环境中的嵌入执行条件**尚未完全满足。下一任开发者优先应排查以下方向。

| 优先级 | 待办 | 说明 |
|---|---|---|
| P0 | 排查 Vercel 运行环境中的嵌入后端选择 | 重点核查 `EMBEDDING_BACKEND`、HF 相关变量、依赖安装与运行时限制。 |
| P0 | 确认 FastEmbed 在 Vercel Python 环境是否可执行 | 若不可执行，需要决定是否改为纯远程 Hugging Face Inference，或改到单独后端。 |
| P1 | 补充正式环境语义记忆回归样例 | 需要验证返回的 `backend_label` 不再是 `local-hash-fallback`。 |
| P1 | 继续收敛 memory API 返回体 | 当前虽然已恢复 200，但 embedding 明细返回较长，可考虑在生产环境默认不返回完整向量。 |
| P2 | 将交付文档与研究结论统一整理进仓库主 README 或 docs | 当前文档散落在多个文件中，后续可集中归档。 |

## 建议的下一步接手顺序

下一任开发者建议先从正式后端开始，而不是先改前端。因为当前用户侧已经可以进入应用，阻塞进一步“完全验收”的关键点，已经集中在**生产环境语义记忆是否真正启用**。

建议顺序如下：首先复查生产环境 `/api/memory/search` 的 `backend_label`；其次核查部署环境变量与依赖是否与本地一致；最后再决定是继续保留当前零成本方案，还是拆分出单独的长期后端承载真正的语义嵌入能力。
